# BlackPanda CRM (React + Node + PostgreSQL)

Полный переезд с версии на HTML+Firebase на архитектуру StitchFlow: React/Vite/Tailwind
на фронте, Express + PostgreSQL на бэке.

## Важно
Этот код не был запущен и протестирован автоматически (в среде подготовки не было
доступа в интернет для `npm install`/подключения к БД). Перед боевым использованием
пройдите шаги ниже и сообщите, если что-то не заведётся с первого раза — пришлите
текст ошибки, поправим.

## Локальный запуск

1. Поднимите PostgreSQL локально (или используйте Railway/Render/Supabase).
2. `cp backend/.env.example backend/.env` и впишите свои данные подключения + `JWT_SECRET`.
3. Установка зависимостей:
   ```
   npm run install:all
   ```
4. Применить схему БД:
   ```
   npm run migrate
   ```
5. Создать первого пользователя (админа):
   ```
   npm run seed
   ```
   По умолчанию: `admin@blackpanda.local` / `changeme123` — обязательно смените пароль
   после первого входа, либо задайте `SEED_EMAIL`/`SEED_PASSWORD`/`SEED_NAME` перед запуском.
6. Запуск в двух терминалах:
   ```
   npm run dev:backend    # http://localhost:4000
   npm run dev:frontend   # http://localhost:5173 (проксирует /api на бэкенд)
   ```

## Деплой (например, на Railway)
`railway.toml` уже настроен: он собирает фронтенд, ставит зависимости бэкенда,
прогоняет миграцию и запускает сервер, который сам раздаёт собранный фронт
(`frontend/dist`) — то есть один сервис вместо двух.

Не забудьте добавить сервис PostgreSQL и переменные окружения: `DATABASE_URL`
(Railway подставит сам, если добавить Postgres-плагин), `JWT_SECRET`.

## Что перенесено из старой (HTML + Firebase) версии
- Модели ноутбуков и серийные номера (склад, статусы: в пути / на складе / резерв / продан)
- Клиенты, их долги
- Предзаказы + передача клиенту со сканированием серийников (аналог confirmPoTransfer)
- Прямые продажи без предзаказа
- Касса (приход/расход, баланс)
- Курс валют + история курса
- Telegram-уведомления — токен бота теперь хранится и используется **только на сервере**,
  а не в браузере, как было в исходной версии
- Авторизация: JWT + bcrypt вместо Firebase Auth

## Новое (по вашему запросу)
- **Роли и права доступа (как в DLE):** admin / staff (сотрудник) / accountant (бухгалтер) / client (клиент).
  Админ в разделе «Админ-панель → Права доступа» настраивает для каждой роли и для каждого
  раздела (склад, клиенты, предзаказы, продажи, касса, настройки) — можно ли его видеть и
  можно ли в нём редактировать. Плюс точечные права на конкретного человека, которые
  перекрывают права его роли (например, один сотрудник — без доступа к кассе, хотя все
  остальные сотрудники видят её).
- **Админ-панель:** создание пользователей с паролем прямо в интерфейсе (без Firebase Console),
  включение/отключение доступа, смена роли и пароля.
- **Роль «клиент»:** у клиента, которому вы создали логин, есть свой урезанный портал (`/portal`)
  — видит только свои предзаказы, покупки и долг, ничего больше.
- **Двуязычность RU/ZH** — переключатель в сайдбаре и на экране входа, сохраняется в браузере.
- **Рассчитано на 10 одновременных пользователей** — пул соединений с БД увеличен (`max: 20`),
  все операции передачи товара идут в транзакциях с блокировкой строки (`FOR UPDATE`), поэтому
  два человека не смогут случайно продать один и тот же серийник дважды.
- **Дизайн под ваш референс:** красно-чёрная палитра, логотип-иллюстрация панды
  (`frontend/public/panda-logo-icon.png` — маленькая иконка, `panda-logo-full.png` — большая
  иллюстрация внизу сайдбара), адаптивная вёрстка сайдбара (иллюстрация масштабируется через
  `aspect-ratio`, а не обрезается фиксированной высотой, как в статичном макете).
- **Реальный график продаж/прибыли по месяцам** и **топ-5 моделей по продажам** на дашборде —
  не бутафория из макета, а настоящие агрегирующие запросы к БД (`GET /api/dashboard`),
  отрисованные через `recharts`.

## Что пока не перенесено (можно добавить по запросу)
- Массовая рассылка сообщений клиентам
- Гарантийные уведомления, старение склада (arrival aging)

## Новые разделы (полноценный функционал, не заглушки)
- **Поставщики** — карточки поставщиков (контакт, WeChat, страна), счётчик поставленных единиц.
  Серийники теперь можно привязать к поставщику (`serials.supplier_id`).
- **Сотрудники** — HR-карточки (должность, телефон, дата найма, зарплата), опционально привязаны
  к учётной записи в системе (`users`). Это отдельная сущность от логинов в Админ-панели —
  не у каждого сотрудника есть доступ к CRM.
- **Финансы** — P&L (выручка / себестоимость / валовая прибыль / расходы / чистая прибыль),
  график по месяцам, разбивка расходов по категориям. Расходы в Кассе теперь можно
  категоризировать (`cash_log.category`) — это и питает разбивку в Финансах.
- **Аналитика** — распределение продаж по брендам (донат-диаграмма), топ-8 клиентов по обороту,
  средний чек, топ моделей по продажам.
- **Отчёты** — отчёт по продажам за период + экспорт в CSV, отчёт по остаткам склада.
- **Импорт** — вставка CSV (через `papaparse`) для массового добавления клиентов и моделей склада,
  с предпросмотром перед импортом.
- **Журнал действий** — аудит ключевых операций (продажа, передача по предзаказу, движение кассы,
  создание клиента/модели/пользователя) — кто, что и когда сделал.

Все новые разделы полностью встроены в систему прав доступа: админ в Админ-панели решает,
какая роль (или конкретный человек) видит и может редактировать каждый из них — точно так же,
как уже работающие Склад/Клиенты/Кассa.

## Функционал, перенесённый из большой HTML-версии (по вашему запросу — всё по списку)

**1. Сканер + визард продажи**
- Раздел «Сканер»: 3 шага — скан серийников (камера через `@zxing/library` или ручной ввод) → выбор клиента → итог с ценами по каждой модели и способом оплаты
- Резервирование товара за клиентом с дедлайном (видно на странице Склада, автоснятие резерва по истечении срока при следующем обращении к API)
- Печать чека после продажи и QR-этикеток серийников (кнопка 🏷️ на карточке модели)

**2. Долги / баланс клиентов / банковские счета / сплит-оплата**
- В карточке клиента: баланс предоплаты (пополнение/списание/обнуление с историей), долги с датой погашения, кнопка «Погасить долг», кнопка «Напомнить» (шлёт в Telegram)
- Банковские счета (Сбер/Альфа/Т-банк) на странице Кассы — ручная корректировка, автоматическое пополнение при продаже
- При продаже — любой способ оплаты: полностью (нал/один из банков), частично (остаток в долг с датой), сплит (часть нал + часть на карту), с баланса клиента

**3. Курс ЦБ РФ + графики аналитики + рассылка + QR/печать**
- На странице Аналитика: график продаж/себестоимости/маржи с переключением периода (7д/30д/1г/по годам), график своего курса против курса ЦБ РФ (кнопка «Обновить курс ЦБ» тянет данные с cbr-xml-daily.ru прямо из браузера)
- Раздел «Рассылка»: отправка сообщения всем клиентам с Telegram (или части), шаблоны сообщений, авто-генерация текста со списком товаров в наличии
- Печать чеков и QR-этикеток

## Что из старой версии сознательно не переносил (можно добавить отдельно)
- Каталог-справочник (Library): предзаполненные списки CPU/GPU/цветов/экранов для автокомплита — сейчас поля просто текстовые
- Гарантийные сроки и уведомления по ним
- Прогноз "на сколько дней хватит склада" и алерт "старый склад 30+ дней"
- Учёт возвратов как отдельного статуса с историей причин
- Экспорт склада/истории клиента в Excel (в Отчётах есть только CSV для продаж)

## Сортировка и перетаскивание (drag-and-drop)
- **Справочник**: бренды, серии и все плоские списки (CPU/GPU/RAM и т.д.) по умолчанию идут по алфавиту;
  новый элемент автоматически встаёт в алфавитную позицию относительно соседей. Любой элемент можно
  перетащить мышкой или пальцем на телефоне (иконка ⠿), новый порядок сохраняется на сервере.
- **Порядок меню**: Админ-панель → вкладка «Порядок меню» — админ перетаскивает разделы в нужном порядке,
  это применяется сразу ко всем пользователям.
- Оба перетаскивания реализованы на pointer events (не HTML5 drag-and-drop), поэтому одинаково
  работают и мышкой на компьютере, и пальцем на телефоне/планшете.
- Разделы, к которым у пользователя нет прав на просмотр, не просто скрыты — они вообще не попадают
  в список для рендера (проверка `can(page, 'view')` в Layout.jsx), то есть их не видно в меню совсем.

## Звук при сканировании + карточка серийника с историей
- В «Сканере»: успешное нахождение серийника — короткий высокий сигнал, ошибка/не найден — низкий сигнал (Web Audio API, без файлов)
- Каждый отсканированный серийник кликабелен — открывает карточку серийника в новой вкладке (не мешает продолжить оформление продажи в текущей)
- Карточка серийника (`/serials/:id`): полные характеристики, статус, гарантия, дата поступления/продажи,
  заметки, переключатель уведомлений гарантии, кнопка «Возврат» (если продан — фиксирует от какого клиента и по какой причине)
- Полная история изменений статуса серийника — кто, когда, что изменил, включая продажи с именем клиента и возвраты
